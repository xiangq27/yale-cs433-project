Day 1 (May 5th, 2016)

08:00-09:00 Breakfast
09:00-10:00 Introduction; Overview of Challenges and Uses Cases
10:00-10:15 Coffee Break
10:15-11:45 From MANET (traditional scheduling/routing) to SDN
Setting: There is a large body of existing work such as routing, scheduling, and distributed optimization in the context of mobile ad hoc networks, for functions such as routing, load balancing, interference management and throughput optimization. It is important that we start with a clear understanding of existing work in the same problem domain.
Key Related Work:
     1. Ad hoc routing
     2. Distributed ad hoc scheduling/optimization
Questions:
     1. Can existing MANET work benefit from the emergence of SDN?
     2. Will SDN learn from the existing MANET work?
Goal:
     1. Review key related work
     2. Discuss issues with potential outcomes:
           Positive: they complement really well and advance the state of art for both
           Negative: SDN has no value
Coordinators: Tony Ephremides (Scheduling) and Kin Leung (Distributed Optimization)
11:45-13:15 Lunch Break
13:15-14:45 Wireless SDN Data Path
Setting: A key benefit of SDN is standardization/whiteboxing of forwarding elements. However, "mainstream" SDN data paths focus mostly on wired networks such as data centers on functions such as fast lookup, queueing for QoS, instead of issues in wireless functions such as interference management, coding/decoding, although there is also much related work on programmable (software defined radio) wireless networks.
Key Related Work:
     1. OpenRadio, GNURadio, WARP, Argos, Big station
     2. OpenFlow, P4, POF
     3. Emerging stateful data path
Questions:
     Is it possible to have more standardized/programable data path in the context of wireless networks in general and wireless mobile networks in particular?
Goal:
     1. Review key related work
     2. Discuss issues with potential outcomes:
           Positive: there is a possibility for data path for military SDN networks
           Negative: not possible
Coordinators: Lin Zhong (WARP, Argos), Manu Bansal and Vyas Sekar

Setting: There is a large body of existing work on mobility, in settings such as host mobility as well as infrastructure mobility. Future Internet architectures such as Mobility First consider mobility as a first-class objective. Mobility will be a crucial factor in the networks that this workshops investigates.
Key Related Work:
     1. Mobility support such as Mobility First in future networks
     2. Mobile SDNs
Questions:
     What roles can SDN play in such networks, in particular, from supporting mobility's perspective?

16:15-16:30 Coffee Break
16:30-18:30 Control Path Architecture and Programming
Setting: A key benefit of SDN is logically centralized control. There is a large body of existing work of SDN control path, but they are mostly in the context of wired networks such as data center networks and wide-area networks (WAN). By control path, we mean the complete spectrum from controller placement, controller fault tolerance, controller programming, and controller-forwarding elements integration. This session focuses more in intra-domain (i.e., the same trust domain).
Key Related Work:
     1. Current wireline SDN controller architectures
     2. Current wireless SDN controller architectures
     3. SDN programming frameworks
Questions:
     1. What are the control path architectures?
     2. How do controllers communicate?
     3. How do controllers and forwarding elements communicate?
     4. What is the programming model?
Goal:
     1. Review key related work
     2. Discuss issues with potential outcomes:
           Positive:
           Negative: not possible

09:00-10:15 Coalition For Wireless/MANET in SDN
Setting: Just as interdomain is crucial to integrate the autonomous systems of the Internet, coalition networks can be essential in many missions settings. To satisfy the requirements of interdomain routing, BGP introduces path-vector routing as the foundation to achieve policy routing. Although there is some preliminary study of interdomain SDN (e.g., using IXPs), the networks that we study can be vastly different.
Key Related Work:
     current interdomain SDN designs
Questions:
